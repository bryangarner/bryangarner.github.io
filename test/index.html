Florida Keys Reef Guide

Data sources:

Florida Geographic Data Library, Data.gov, Noaa.gov

Development libraries:

Leaflet, Panolens, jQuery, JavaScript, HTML, CSS

Software used:

Brackets, GitHub, ArcGIS Desktop

Completed August 2019


The goal of this project was to create an interactive web map that allowed users to explore well known and established coral reef sites within the Florida Keys National Marine Sanctuary. The interactive map was an effort to modernize the many static map resources that can be found scattered across the internet. One key aspect of the modernization was to incorporate a mechanism to allow users to view each reef site or marine sanctuary zone in a 360-degree environment. In addition, give the users an opportunity to read information about said reef. The intended audience for this web map ranges from first time reef explorers to committed environmentalists. This web map is nested in a larger pseudo organization’s website called the Florida Keys Reef Guide. This web page provides users with condensed, organized, tips that at a minimum if abided by, will ensure the reefs will be better taken care of for the future. The webpage also includes links to organizations whose primary mission is to care for and educate about coral reefs and coral reefs health.

Users can search for specific features by name or click any feature to get more infomration pertaining to the area. Users also have the ability to view reefs and marine zone areas in a 360-degree environment.

The map was developed with Leaflet for the mapping component along with Panolens for the 360-degree photo viewer. Additional leaflet plugins were also used for the search and the baselayer toggle functions.

Nitrates and Cancer in Wisconsin

Data sources:

Wisconsin Department of Natural Resources

Development libraries:

Leaflet, Turf, Simple Statistics, jQuery, JavaScript, HTML, CSS

Software used:

Brackets, GitHub, ArcGIS Desktop

Completed June 2019


The goal of this project is to use Inverse Distance Weighted spatial interpolation and Ordinary Least Squares linear regression to examine the relationship between nitrate concentrations in water wells and cancer rates in the state of Wisconsin.

Users can use the slider controls to select a distance decay weight to use for the spatial interpolation and the hexbin size in kilometers.

The map was developed with Leaflet, using Turf for the geoprocessing and Simple Statistics to process the linear regression.

Lake Manatee State Park Visitor Map

Data sources:

Florida Department of Environmental Protection

Development libraries:

Leaflet, jQuery, JavaScript, HTML, CSS

Software used:

Brackets, GitHub, ArcGIS Desktop CARTO

Completed July 2019


The goal of this project was to create a mobile-friendly web application for tourists visiting the Lake Manatee Sate Park.

This application was built with four main features, the ability for users to bookmark points within the park, geolocate their device, submit a comment to the database, and filter the comments on the map. The data is hosted on a CARTO database being manipulated using php and sql.

The map was developed with Leaflet.

Housing and Living by Florida Counties

Data sources:

University of Wisconsin County Health Rankings 2016, Census Bureau ACS 5-year Estimate 2016

Development libraries:

D3, jQuery, JavaScript, HTML, CSS

Software used:

Brackets, GitHub, Microsoft Excel

Completed April 2019


This is an overview of data pertaining to housing and living in Florida by county looking at commuting, property values, homeowner percentages and housing problem data.

D3 allows you to bind arbitrary data to a Document Object Model (DOM), and then apply data-driven transformations to the document.

This was an introductory project to the D3(data driven documents) library.

Tour Of Chile

Data sources:

Natural Earth, Viewfinder Panoramas, UN Food and Agriculture Organization

Software used:

ArcGIS Desktop, Adobe Photoshop, Adobe Illustrator

Completed Oct 2018


This print map was created as part of a graphic design in cartography course. The challenge for this course included planning and digitizing a real-world route, and then combining it with a terrain basemap that fit with the challenge scenario.

The vector post-processing was conducted in Illustrator. In order to make it more realistic and appealing, Photoshop was used to adjust the terrain features, tweak colors and add textures according to the land cover.

Geogander

Software used:

Adobe Photoshop, Adobe Illustrator, Adobe After Effects, InVision Studio

Completed Sept 2017


This is a prototype created to visualize a proposed geospatial app called Geogander. The idea behind this project is to rapidly view a simulated end product to get the look, feel, and idea across to an audience.

Mock images and animations were created using Illustrator and Photoshop, later imported into InVision Studio to simulate a mobile application environment. Recorded with QuickTime and edited using Adobe After Effects.
